# Security Policy

## Приватность

blockcheckw **не собирает телеметрию**, не отправляет данные на внешние серверы и не логирует
домены, IP-адреса или DNS-запросы пользователя.

Все данные остаются локально:
- JSON-отчёты сохраняются в текущую директорию
- Конфигурация хранится в `~/.config/blockcheckw/`
- nfqws2 запускается без внешних подключений (только NFQUEUE)

## nfqws2

nfqws2 — сторонний бинарь из [zapret2](https://github.com/bol-van/zapret2). blockcheckw
управляет его жизненным циклом (запуск/остановка), но не контролирует его внутреннее
поведение. По вопросам безопасности nfqws2 обращайтесь в upstream-проект.

## Сообщить об уязвимости

Если вы нашли уязвимость в blockcheckw:

1. **Не** создавайте публичный issue
2. Напишите на email: **redtom@yandex.ru** с темой `[SECURITY] blockcheckw`
3. Приложите описание, шаги воспроизведения и (если возможно) патч

Ответ — в течение 72 часов.

## Поддерживаемые версии

Исправления безопасности выпускаются только для **последней** версии.
